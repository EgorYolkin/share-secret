# share-secret

[English](./README.md) | [Русский](./README.ru.md)

`share-secret` шифрует текст в `.txt` файл, генерирует случайный пароль, выводит его в терминал и показывает файл в Finder.

## Возможности

- Одна команда для создания зашифрованного `.txt` файла
- Генерация случайного пароля через `openssl`
- Вывод пароля в терминал
- Показ файла в Finder (`open -R`)
- `share-secret setup` для YAML-конфига (user/global)

## Требования

- macOS
- `openssl`

## Использование

```bash
share-secret "мой секретный текст" "my-secret.txt"
```

Аргументы:

- Первый аргумент: текст для шифрования (обязательный)
- Второй аргумент: имя выходного файла (необязательный, по умолчанию: `secret.txt`)

Результат:

- Зашифрованный файл
- Пароль в выводе терминала

Путь по умолчанию:

- Текущая директория, из которой вызвана команда

## Команда настройки

```bash
share-secret setup --user --output-dir ~/Secrets
```

Опции:

- `--user` записать конфиг в пользовательскую config-папку (по умолчанию)
- `--global` записать конфиг в системную config-папку (нужен `sudo`)
- `--output-dir <path>` директория для зашифрованных файлов (`.` = текущая директория)

Конфиг хранится в YAML (с запасом под будущие параметры):

```yaml
output_dir: "."
```

Приоритет конфигов:

- Сначала user config, потом global config
- Если конфигов нет, используется текущая директория

## Локальная установка

Из корня репозитория:

```bash
./scripts/install.sh
```

## Быстрая установка из GitHub

```bash
curl -fsSL https://raw.githubusercontent.com/<YOUR_GITHUB_USER>/share-secret/main/scripts/install-from-github.sh | bash
```

## Заметки по безопасности

- Используется шифрование `AES-256-CBC` с `PBKDF2` и случайной солью.
- Сохраните выведенный пароль безопасно: без него расшифровка невозможна.
